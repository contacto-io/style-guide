# Welcome to Contacto Components Library

Storybook helps you build UI components in isolation from our app's business logic, data, and context.
That makes it easy to develop hard-to-reach states. These **stories** are different states of a component which can be independantly developed and tested.

<div className="subheading">Components</div>
The following are the common components that are built. Each of them, has stories, which define their UI states. You can play with it and use it as you wish.

1. [Buttons](?path=/docs/components-button--primary)
2. [Checkbox](?path=/docs/components-checkbox--default)
3. [Radio](?path=/docs/components-radio--default)
4. [Select](?path=/docs/components-select--default)
5. [TextField](?path=/docs/components-textfield--default)

<div className="subheading">How to use?</div>

1. Import the component that is required.
2. Visit the particular component page.
3. Navigate to your required story.
4. Copy the code and modify the `props` as required.
5. Since all the components are wrapper components upon antd. Please visit their docs for additional information.

<div className="subheading">Technical Requirements</div>

For the compoenents to work properly, make sure the consumer project has these dependencies installed. The following are the peer dependencies for this component library.

```json
  "peerDependencies": {
    "react": "^16.13.1",
    "react-dom": "^16.13.1",
    "prop-types": "^15.7.2",
    "antd": "^4.16.8"
  }
```
